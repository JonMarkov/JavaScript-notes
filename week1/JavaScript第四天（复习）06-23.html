基本数据类型和引用数据类型的区别
堆栈内存：在浏览器加载页面时，浏览器会形成两个虚拟内存；
栈内存：提供代码运行环境和存储基本数据类型的值；
堆内存：存储引用数据类型的值；
基本数据类型存储到栈内存中，引用数据类型存储到堆内存中；
基本数据类型是操作值的，引用数据类型操作的空间地址；

boolean (布尔类型)
布尔类型有且只有两个值：true和false;
Boolean：把其他数据类型的值转换成布尔类型，结果只有true和false（js中有切只有5个值转布尔是false：0 "" NaN undefined null）;
逻辑运算符：
    1，|| ：或的意思；
        在条件语句中，||两边至少有一个为true，整体结果就是true；
        在赋值的情况下，如果||前面值转布尔是true，直接取这个值；如果是false，直接取后面的值；例（var a = ""||NaN） a = NaN ）
    2，&& ：并且的意思，只要有一个是false，整体结果就是false；
        在赋值情况下，如果&&前面的值转布尔是false，直接取这个值，如果是是true，直接去后面的值；例（var a = ""&&NaN） a = "" ）
    3,！：取反；首先会把！后面的值转布尔，然后在取反；
    4，！！：取反在取反，返回值是直接取布尔值；
null和undefined（null和undefi数据类型）
值类型：没有
null的常见情况：
    1，当通过id获取元素时，如果id不存在，那么结果就是null；
    2，正则捕获时，如果没有捕获到内容，结果就是null；
    3，当对象被清空对象时；
undefined的常见清空：
    1，如果变量只声明，没有赋值，那么默认存储就是undefined；
    2，函数如果有形参，没有实参赋值，那么形参默认存储是undefined；
    3，函数如果没有return，那么函数默认返回值是undefined；
    4，获取对象属性名对应的属性值，如果属性名不存在，那么返回值是undefined

数据类型之间的比较

1、= ：赋值
2、== ：比较；数据类型之间的比较；返回值是一个布尔值 （true或false）；
数据类型之间比较时，会默认进行数据类型之间的转换，然后再进行比较；
3、数据类型比较时转换的规律：
    对象 == 对象 --> 比较的是空间地址；
    对象 == 字符串 --> 首先把对象转换成字符串，然后再进行比较；
    对象 == 布尔 --> 对象首先调用toString转换成字符串，然后在调用Number转数字；布尔直接转数字，进行比较；
    对象 == 数字 --> 对象首先调用toString转换成字符串，然后在调用Number转数字；然后和数字进行比较；
    （NaN同任何比较都是false，包括NaN）
    布尔 == 字符串 --> 布尔转数字，字符串转数字，然后进行比较；
    null == undefined -->  true
4、=== ：绝对比较，不会进行数据类型之间的转换，
5、！：（！值）放在值的前面取反；  （！=）放在等号前面是不等于；
6、！=：会进行数据类型之间的比较；（在表达式两边的数据类型不一致时,会隐式转换为相同数据类型,然后对值进行比较.
7、！==：绝对不等于(不会进行类型转换,在比较时除了对值进行比较以外,还比较两边的数据类型,===的非形式)
    简单理解就是 != 会转换成相同类型 进行比较,!== 除了比对值还比对类型
