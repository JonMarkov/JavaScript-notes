this

this是一个关键字，它代表当前函数的执行主体；
this只代表值，不能赋值（this的代表值取决于应用场景，场景不同，this的值就不同）
this的用法
1、this在全局作用域下使用，它代表全局对象window；
2、自执行函数中的this全部都是window；
3、函数作为参数的时候，一般情况下this都是window；
    （其他情况，像forEach  map...传递的第二个参数可以修改this指向）
4、元素事件上的函数，this一般情况都是当前元素
5、函数执行，看函数的执行主体前面是否有点，如果有，点前面是谁，this就是谁，没有点，this都是window

单例模式
定义：把描述同一功能的属性和方法放到一个对象「命名空间」中，这种模式叫做单例模式；

在设计模式中，这个person1不仅仅是对象名，它是一个命名空间，多个命名空间是相互独立的，互不干扰；
优点：不受全局变量的污染和干扰，并且可以相互调用方法；
表现形式
let person1 = {
    name : 'qw',
    home:function(){
         person2.login();
    }
   
}

let person2 = {
    name : 'nth',
    login:function(){
        
    }
}
由来
单例模式本身就是一个对象，它Object类上的一个实例，实例与实例之间互相不干扰，叫做单独的实例，简称“单例”
高级单例模式
定义：在高级单例模式当中，我们不只会把一个对象赋值一个命名，我们会让它先执行一个自执行函数
「就形成了一个闭包，一个不销毁的作用域」，在自执行函数中，返回一个对像给这个命名空间

优点：我们可以在这个私有的作用域中，创建很多属性和方法，仅需要把我们使用的属性和方法暴露在这个对象当中即可；

面向对象

OOP/OP:面向对象的缩写,面向对象思想；
JavaScript就是通过面向对象的思想设计开发出来的；
定义：面向类的封装、继承和多态，通过简单实例话，调用其方法和属性；
学习面向对象，需要了解，对象、类、实例；
对象：
万物皆对象，它多为一种泛指，比如我们常说的研究对象，这个对象可以是任何物体
类 ：
是对象中具有同一属性和特征的集合，我们称之为类，类又分为大类和小类；
（举例：世间万物凡是有生命的物体，我们都可以叫做生物类，生物类下又分为动物类、植物类、人类；人类下又分为男人和女人类）
实例：
实例是类中的一个具体的细分；我们研究当前实例具有哪些属性和方法，那么证明当前类中的其他实例也具有同样的属性和方法；

JavaScript中的内置类

在JavaScript中任何数据类型都有它所属的类，除了null和undefined
数据类型：Number  String   Boolean   object   Array   RegExp(正则)   Function   Date  ...
类的首字母都是大写
类的执行都是通过new来执行
Object对象类，被称为基类，任何数据类型都可以通过__proto__「原型链」来找到object基类
字面量方式的形式来创建

通过实例化的方式来创建（通过实例化创建出来的值，它是用对象包裹的，和字面量方式创建表现形式不同而已，功能方法都一样）
    var num = new Number(1)
基本数据类型方式的创建：
    var num = new Number(1)
    var boo = new Boolean()
    var str = new String("1")
引用数据类型方式的创建：
    var ary = new Array(1,2)/Array(9)
    var obj = new Object({})
    var fn = new Function("a","b","return 1")
    (创建函数：如果参数只有一个的话，这个擦数就属于函数体的内容，如果是2个或者2个以上的话，最后一个是函数体，其余的都是形参)

构造函数

定义：当一个函数通过new关键字来执行的时候，这个函数就不是普通函数了，它是一个构造函数，也是一个自定义类，当前的函数名，就是类名，这个函数的返回值就是类的实例；

**为了让构造函数和普通函数有一个区别，我们建议写构造函数的时候函数名首字母大写**

注意：一般情况下写业务逻辑的时候不会用到构造函数，但是在封装库、框架、插件以及组件的时候会用到构造函数模式；

构造函数执行的时候，如果没有参数的时候，小括号可以不用写；

构造函数的执行过程

普通函数的this一般指向是window，构造函数this指向是实例；
普通函数过程：
    1、形成一个私有作用域；
    2、形参赋值
    3、变量提升
    4、代码从上到下执行
    5、判断作用域是否销毁；
构造函数的执行过程：
    1、形成一个私有的作用域
    2、形参赋值
    3、变量提升
    4、浏览器会创建一个对象「开辟了一个新的堆内存」将这个对象指向了this「堆内存指针指向this实例」，this就是实例；（this是当前函数的返回结果）
    5、代码从上到下执行
    6、判断当前构造函数是否有return，如果没有return默认将实例返回，如果有return：（当return的是基本数据类型时，对实例没有影响；如果是引用数据类型，那么实例就是该引用数据类型）
    （建议，不要轻易return引用数据类型）
私有属性
在构造函数中，给this添加的属性值和方法，都属于当前实例的私有属性
公有属性
在构造函数中，当前实例中，通过__proto__找到的所有属性和方法都是公有属性

补充：实例想要调取公有属性，直接.就可以调取「底层就是通过__proto__去找这个属性」

注：用in这种方式判断当前属性名「公有+私有」是否属于这个属性；

Object类提供一个hasOwnProperty，这个方法判断当前属性是否是该实例的私有属性；

封装一个判断是否是他公有属性的方法：
person.__proto__.hasPubProperty = function(){
    return key in this && !this.hasOwnProperty(key)
}

原型模式

在JavaScript中
函数分为：
    1、普通函数；
    2、类「自定义类和内置类」；
    3、函数类的实例；
对象分为：
    1、普通对象「对象数据类型」；
    2、构造函数new出来的一个实例，也是一个对象；
    3、类上面的原型也是一个对象；
    4、函数也是一个对象「万物皆对象」；
学习原型模式需要记住三句话
1、所有的函数都天生再带一个属性，叫做prototype「原型」，他是一个对象「既然是对象，那么就是一个堆内存」；
2、所有类的原型上面「就是开辟的这个堆内存」，都天生自带一个属性，叫做constructor「构造函数」，它指向当前类本身；
3、所有的对象都天生自带一个属性__proto__,他指向当前所属类的原型

原型链

一个实例要找一个属性，优先会去找自己的私有属性，如果自己的私有属性没有的话，会通过__proto__找到自己所属类的原型上面「公有属性」，
如果公有属性还没有，继续通过__proto__找到自己所属类的原型，直到找到Object「基类」的原型上，一直找到基类还没有的话。直接返回undefined；
